## 1. 项目背景与目标
### 1.1 背景
传统英语单词背诵方式（如死记硬背）对儿童而言枯燥乏味，且缺乏视觉联系和发音语境，导致“记得快、忘得快”。
### 1.2 目标
通过 **Google Gemini AI** 实现个性化内容生成，结合 **“五步循环学习法”** 和 **“节奏强化模式”**，打造一款沉浸式、高反馈的拼写学习工具，帮助儿童建立单词的“音-形-义”深度连接。

---

## 2. 产品功能架构 (Functional Architecture)

### 2.1 用户管理层
- **多账户体系**：支持创建多个独立用户，每个用户拥有独立的 `userId`、学习进度、统计数据和偏好设置。
- **登录鉴权**：支持用户名+密码登录，默认密码为 `123`。
- **API Key 管理**：支持用户自定义 Gemini API Key，或使用系统默认 Key。

### 2.2 核心逻辑层 (AI & 算法)
- **单词验证引擎**：利用 AI 过滤非实义词，确保学习内容的有效性。
- **多模态生成引擎**：
    - **文本生成**：生成音标、翻译、例句、关联词、分块 (Chunks) 及 TTS 助记发音。
    - **图像生成**：生成与单词语义高度匹配的卡通风格插图。
- **模糊匹配算法**：在语音识别和拼写校验中使用 Levenshtein 距离算法，允许合理的微小误差。

### 2.3 教学执行层 (五步法)
- **Observe (观察)**：视觉输入 + 影子跟读 (Shadowing)。
- **Listen (聆听)**：听觉辨析 + 词根/助记词记忆。
- **Practice (练习)**：多维度互动（选择、填空、排序）。
- **Test (测试)**：闭卷拼写挑战。
- **Rhythm (节奏)**：动感节奏下的肌肉记忆强化。

### 2.4 数据持久层
- **IndexedDB 本地库**：存储用户信息、单词记录 (`words`) 和每日统计 (`daily_stats`)。
- **备份系统**：支持数据加密导出与校验导入。

## 3. 用户使用流程 (User Flow)

1.  **启动与登录**：用户进入应用，选择已有档案或创建新档案。
2.  **单词输入**：用户通过语音（麦克风）或键盘输入想要学习的单词。
3.  **AI 预处理**：系统调用 Gemini 进行单词验证。若合法，则并行生成单词数据和插图。
4.  **进入学习循环**：
    - **Step 1 (Observe)**：查看图片和翻译，点击分块听发音，进行语音跟读。
    - **Step 2 (Listen)**：听单词发音，阅读 AI 生成的趣味助记词。
    - **Step 3 (Practice)**：完成三个子环节：从干扰项中选出正确分块、填入缺失字母、将乱序分块排序。
    - **Step 4 (Test)**：将字母磁贴拖拽到正确位置完成拼写。
    - **Step 5 (Rhythm)**：开启节奏模式，随节拍点击正确字母。
5.  **结算与反馈**：展示获得的星星，更新每日统计，查看勋章。

## 4. 导航设计与核心页面跳转逻辑 (Navigation & Transition Logic)

### 4.1 全局导航 (Bottom Navigation)
应用采用底部导航栏作为一级导航，包含四个核心入口：
- **首页 (Home)**：默认入口。包含单词输入、快速开始学习、每日挑战入口及今日概览。
- **节奏 (Rhythm)**：**每日挑战 (Daily Challenge)** 模式入口。点击后直接进入当日学习单词的节奏挑战。
- **词库 (Library)**：展示用户已学习的所有单词。支持按日期筛选、查看单词详情、发起历史单词挑战及导出备份。
- **统计 (Stats)**：展示用户的学习成就，包括总星星数、总勋章数、最高 BPM 及每日学习曲线。支持发起**随机模式 (Random Mode)** 挑战。

### 4.2 核心跳转逻辑 (State Machine)
应用通过 `GameStep` 状态机控制视图切换：

#### A. 单词学习流 (Learning Loop)
1. **输入 (INPUT_WORD)** → 用户输入单词并由 AI 验证。
2. **观察 (STEP_1_OBSERVE)** → 展示 AI 生成的图片、翻译及分块。
3. **聆听 (STEP_2_LISTEN)** → 播放发音并展示助记词。
4. **练习 (STEP_3_PRACTICE)** → 包含分块选择、字母填空、分块排序三个子环节。
5. **测试 (STEP_4_TEST)** → 闭卷拼写测试。
6. **成功/失败 (SUCCESS/FAIL)** → 结算页面，成功后可选择进入“节奏模式”或返回首页。

#### B. 节奏挑战流 (Rhythm Flow)
1. **入口 (RHYTHM_INTRO)** → 选择模式（每日/随机），展示当前 BPM 和规则。
2. **挑战 (STEP_5_RHYTHM)** → 进入游戏环节。
3. **结算** → 完成后返回“节奏入口”或“首页”。

### 4.3 页面切换规范
- **平滑过渡**：所有页面切换均需配合 `framer-motion` 的 `AnimatePresence` 实现淡入淡出或滑入效果。
- **状态持久化**：在学习流中，若用户意外退出，系统应尝试保留当前单词的进度（通过 `IndexedDB` 暂存）。
- **返回逻辑**：在非学习流页面（词库、统计），点击底部导航可直接切换；在学习流中，点击返回需弹出确认框以防进度丢失。

## 5. 核心策略描述 (Detailed Strategies)

### 5.1 单词生成策略 (Gemini Prompt Engineering)
系统采用多阶段生成流程，确保单词内容的准确性、教育性和趣味性。

- **模型选择**：
    - **文本/逻辑**：使用 `gemini-3-flash-preview`（低延迟，强逻辑）。
    - **图像生成**：使用 `gemini-2.5-flash-image`（儿童插画风格）。
- **词性限制与过滤 (POS Restrictions)**：
    - **允许词性**：仅限 **名词 (Nouns)**、**动词 (Verbs)** 和 **形容词 (Adjectives)**。这些词类具有明确的语义和视觉关联性，适合儿童学习。
    - **禁止词性**：副词、代词、介词、连词、冠词、叹词、助词。
    - **验证流程**：在生成数据前，系统会先进行词性判定。若不属于允许类别，将拦截并提示用户输入更具象的单词。
- **分块逻辑 (Chunking - "Right-to-Left" Strategy)**：
    - **核心逻辑**：采用“从右至左”分析法，将单词拆解为可发音的拼写块（Spelling Chunks）。
    - **规则**：
        - **元音导向**：从右向左扫描，一个元音音素与其前导辅音组合为一个块（如 `tiger` -> `ti-ger`）。
        - **Silent E**：词尾不发音的 `e`（如 `cake`）不单独成块，归入前一组合。
        - **元音组合 (Vowel Teams)**：双元音（如 `ai`, `ea`, `oa`）视为一个元音音素。
        - **后缀保护**：常见后缀（如 `ment`, `tion`, `ing`）保持完整。
        - **单音节保护**：仅含一个元音音素的单词（如 `bird`, `shirt`）不进行拆分。
    - **目标**：每个块都应是可拼读的，且符合“辅音领头”的自然发音流。
- **TTS 优化 (Mnemonic Spelling)**：
    - **痛点**：TTS 引擎在单独朗读分块时（如 `ti`）可能出现发音偏差。
    - **方案**：AI 为每个分块生成“助记拼写”（如 `ti` -> `tie`），确保在 Step 3 的分块点击环节发音准确。
- **记忆辅助 (Root/Mnemonic)**：
    - 避开复杂的拉丁语源，采用基于谐音或简单单词组合的“儿童版”记忆法（如 `purple` -> `purr (like a cat) + ple (sounds like pull)`）。

### 5.2 拼写游戏策略 (Spelling Game Strategy)
应用通过多层级的互动环节，逐步引导儿童从“识别分块”过渡到“完整拼写”。

#### A. 练习环节 (Step 3: Practice)
包含三个子环节，难度递增：
1.  **分块选择 (CHUNK_SELECT)**：
    *   **规则**：从单词中随机隐藏一个分块，提供 1 个正确选项和 3 个干扰项。
    *   **干扰项生成逻辑**：
        *   **元音替换**：将分块中的元音替换为其他元音（如 `ple` -> `pla`）。
        *   **后缀/前缀修改**：增加或减少一个字母（如 `ing` -> `in`）。
        *   **镜像反转**：将分块字母顺序反转（如 `ap` -> `pa`）。
        *   **字符倍增**：重复分块中的某个字母（如 `cat` -> `catt`）。
2.  **分块填空 (LETTER_FILL)**：
    *   **规则**：隐藏一个随机分块，用户需通过键盘输入该分块的完整拼写。
3.  **分块排序 (CHUNK_SORT)**：
    *   **规则**：将单词的所有分块打乱顺序，用户需按正确顺序点击分块以重组单词。

#### B. 测试环节 (Step 4: Test)
*   **规则**：进入“闭卷”模式，单词被拆解为单个字母磁贴。
*   **交互**：用户需将字母磁贴按正确顺序填入槽位。
*   **防错机制**：拼写错误时触发 `Dissonance` 音效和屏幕晃动，并高亮显示错误位置，但不中断流程，鼓励用户再次尝试。

### 5.3 节奏游戏策略 (Rhythm Mode)
- **双模式设计**：
    - **每日挑战 (Daily Challenge)**：以当日学习的单词为题库。成功后 BPM 永久提升 5，挑战难度随天数递增。
    - **随机模式 (Random Mode)**：从历史词库中随机抽取 5 个单词（排除今日新词）。BPM 重置为 80，适合复习和热身。
- **动态 BPM**：初始 80 BPM。在每日挑战中，连续成功后 BPM 逐级提升（最高可达 140+），增加挑战性。

### 5.4 奖励与统计策略
- **星星 (Stars) 计算**：
    - 完成一次单词学习 (Step 1-4) 并通过测试：+1 颗星。
- **勋章 (Badges) 授予**：
    - 每完成一次完整的节奏挑战 (Step 5)：+1 枚勋章。
    - 勋章在“统计”页面以累计总数形式展示，象征用户的坚持与节奏掌控力。
- **BPM (速度) 提升**：
    - 节奏挑战成功后，下一次挑战的 BPM 会自动提升 5，增加难度。
- **统计维度**：记录 `highestBpm`（最高速度）、`successCount`（成功单词数）、`totalTime`（学习总时长）、`stars`（总星星数）和 `badges`（总勋章数）。

### 5.5 多帐户体系策略 (Multi-Account Strategy)
系统支持多用户并行使用，确保家庭或教室场景下的数据隔离与个性化体验。

1. **角色定义**：
    - **Eva (管理员/默认用户)**：
        - 拥有最高权限，可进入“用户管理”面板。
        - 可查看所有用户的 ID、当前明文密码（用于找回）。
        - 可强制重置其他用户的密码。
        - 可导出包含所有用户凭证的 `users_credentials.json` 文件。
    - **普通用户**：
        - 仅能访问自己的单词库和统计数据。
        - 无法查看其他用户信息。
2. **数据隔离机制**：
    - **存储层**：所有 IndexedDB 存储（`words`, `daily_stats`）均以 `userId` 作为复合主键的一部分。
    - **逻辑层**：应用启动时根据 `currentUser` 过滤所有列表。切换用户会触发全局状态重置，确保数据不串行。
3. **帐户切换流程**：
    - 用户通过顶部导航栏的头像下拉菜单进行切换。
    - 切换时需输入对应账户的密码（若已设置）。
    - 切换成功后，系统自动重新加载该用户的学习进度和个性化设置。
4. **安全性与隐私**：
    - **本地加密**：非 "Eva" 用户的导出数据强制进行 XOR 混淆，防止直接读取 JSON 内容。
    - **密码保护**：支持为每个账户设置独立密码。默认用户 "Eva" 的初始密码为 `123`。
5. **数据迁移与备份**：
    - 支持“全量账户导出” (Account Export) 和“仅单词导出” (Words Export)。
    - 导入时系统会校验 `userId` 或 `username`，防止将 A 用户的数据错误导入 B 用户的库中。

## 6. 数据结构与导入导出规则

### 6.1 离线单词数据 JSON 结构 (Schema)
```json
{
  "exportDate": "2024-01-01T00:00:00Z",
  "exportType": "words | account",
  "username": "Eva",
  "words": [
    {
      "userId": "user_123",
      "word": "apple",
      "data": {
        "word": "apple",
        "partOfSpeech": "n.",
        "parts": ["ap", "ple"],
        "partsPronunciation": ["ap", "pull"],
        "translation": "苹果",
        "sentence": "I eat a red apple.",
        "imageUrl": "data:image/png;base64,..."
      },
      "dateAdded": "Mon Jan 01 2024",
      "lastReviewed": "Tue Jan 02 2024",
      "bestTime": 12.5
    }
  ],
  "stats": [] // 仅在 account 类型导出时包含
}
```

### 6.2 导入导出规则
- **加密规则**：
    - 非 "Eva" 用户的备份文件在导出时会经过 **XOR 混淆**（Key: `star-speller-secret-key`）并进行 Base64 编码。
    - 导入时系统会自动检测是否需要解密。
- **冲突处理**：
    - **覆盖模式**：若导入的单词已存在，以导入文件中的数据为准。
    - **合并模式**：保留现有的学习记录，仅补充缺失的单词。
- **安全性校验**：
    - 导入 `account` 类型文件时，系统会校验文件名中的用户名与当前登录用户是否匹配，防止误导入他人数据。
    - 自动清洗 (Sanitize) JSON 字符串中的非法控制字符，防止解析崩溃。

### 6.3 单词数据复用规则 (Data Reuse Rules)
为了减少 API 调用次数、节省 Token 消耗并提升用户体验，系统遵循以下数据复用逻辑：

1. **本地优先 (Local Cache First)**：
    - 当用户输入单词时，系统首先检索当前用户的 IndexedDB 记录。
    - 若存在匹配记录，直接复用其分块、发音、翻译及图片数据，不触发 AI 生成。
2. **跨用户复用 (Cross-User Reuse)**：
    - 若当前用户库中无此单词，系统会检索本地数据库中**其他用户**已生成的单词数据。
    - 若找到匹配项，系统将克隆该数据（包括分块逻辑和图片）至当前用户名下，仅更新 `userId` 和 `dateAdded`。
3. **数据补全机制 (Data Completion)**：
    - 复用数据时，若发现关键字段缺失（如 `imageUrl` 为空），系统将仅针对缺失部分调用 AI 生成（如 `generateWordImage`），并更新数据库。
4. **版本迁移 (Schema Migration)**：
    - 当 `WordData` 结构发生变更（如增加 `partsPronunciation` 字段）时，系统在初始化时会自动运行迁移脚本，对旧数据进行补全或格式转换。
5. **手动刷新 (Manual Refresh)**：
    - 用户在学习界面可手动触发“重新生成图片”或“纠正分块”，此时系统将忽略缓存，强制调用 AI 生成最新数据并覆盖旧记录。
6. **预置词库 (Seeding)**：
    - 新用户初始化时，系统会自动从 `INITIAL_WORDS` 预置词库中导入基础单词，避免冷启动时的等待。

## 7. UI/UX 设计需求 (UI/UX Design Requirements)

### 7.1 视觉风格与情绪 (Mood & Style)
- **核心情绪**：活泼 (Playful)、充满活力 (Energetic)、亲和力 (Child-friendly)。
- **设计语言**：采用大圆角、粗边框和高对比度的色彩组合，营造出一种“数字玩具”的质感。

### 7.2 色彩规范 (Color Palette)
- **主色 (Primary)**：`#3B82F6` (Vibrant Blue) - 用于主按钮和核心品牌色。
- **辅助色 (Secondary)**：`#FB923C` (Sunny Orange) - 用于强调、星星和勋章。
- **成功色 (Success)**：`#22C55E` (Leaf Green) - 用于正确反馈。
- **背景色 (Background)**：`#F8FAFC` (Soft Slate) - 干净的浅色背景，突出内容。

### 7.3 字体规范 (Typography)
- **标题与品牌**：`Fredoka` (Sans-serif) - 圆润、加粗，极具亲和力。
- **正文与数据**：`Inter` - 确保在小字号下的高可读性。
- **字号阶梯**：
    - 巨型标题：6xl (用于品牌名)。
    - 模块标题：2xl (用于卡片标题)。
    - 辅助说明：xs (用于微型标签)。

### 7.4 组件设计规范 (Component Specs)
- **容器 (Containers)**：
    - 圆角：统一采用 `2.5rem` (40px) 的超大圆角。
    - 边框：关键容器使用 `4px` 的实色边框，模拟实体玩具的厚重感。
- **按钮 (Buttons)**：
    - 交互：Hover 时轻微放大 (scale-105)，点击时缩小 (scale-95)。
    - 阴影：使用深色投影 (Shadow-xl) 增加层级感。
- **触控优化**：所有可点击区域的最小尺寸不低于 `44x44px`，确保儿童在平板上的操作体验。

### 7.5 动效与反馈 (Motion & Feedback)
- **进入动效**：使用 `framer-motion` 实现组件的弹簧式 (Spring) 进入效果。
- **负面反馈**：拼写错误时，容器触发 `Shake` (左右晃动) 动画，并伴随 `Dissonance` 音效。
- **正面反馈**：成功时触发粒子喷泉效果 (Particle Burst) 和 `Harmony` 音效。
- **节奏感**：在节奏模式下，字母需随背景音乐的节拍进行有规律的缩放跳动。

### 7.6 核心交互策略 (Core Interaction Strategies)

#### A. 语音输入交互 (Voice Input)
- **交互逻辑**：
    - **长按触发 (Hold-to-Speak)**：采用“按下开始、松开结束”的逻辑，符合直观操作习惯。
    - **震动反馈 (Haptic Feedback)**：开始录音时触发短震 (50ms)，结束录音时触发微震 (30ms)，提供触觉确认。
    - **自动纠错 (Auto-Correction)**：利用 Levenshtein 距离算法对识别结果进行模糊匹配，允许轻微的发音偏差。
- **视觉反馈**：
    - **动态波纹 (Voice Waves)**：麦克风按钮周围出现多层半透明的扩散圆环（`animate-ping`），模拟声波扩散效果。
    - **状态指示器 (Status Indicator)**：按钮上方弹出悬浮标签（如 "I'M LISTENING! 👂"），并伴随跳动动画。
    - **全屏聚焦 (Focus Effect)**：录音时屏幕边缘出现淡淡的红色呼吸灯效果（Glow），引导用户专注于说话。
    - **实时预览 (Live Transcript)**：在输入框中实时展示识别到的文字片段，增强掌控感。
- **容错与引导**：
    - **无声检测**：若录音超过 5 秒无有效输入，自动停止并弹出提示“没听清，请再试一次”。
    - **权限引导**：若麦克风权限被禁用，展示清晰的图文引导，告知用户如何在浏览器设置中开启。

#### B. 节奏游戏交互 (Rhythm Game)
- **视觉节拍提示 (Visual Cues)**：
    - **收缩环 (Shrinking Rings)**：每个目标字母外围有一个随节拍收缩的圆环，当圆环与字母边界重合时即为最佳点击时机（Perfect）。
    - **跳动动效 (Pulse)**：字母随背景鼓点（Kick）同步跳动，增强节奏代入感。
- **判定与反馈 (Judgment & Feedback)**：
    - **Perfect (完美)**：在节拍点前后 50ms 内点击。触发金色粒子喷泉，显示 "PERFECT!" 字样，Combo 计数器剧烈跳动。
    - **Good (优秀)**：在节拍点前后 50ms-150ms 内点击。触发蓝色火花，显示 "GOOD" 字样。
    - **Miss (错过)**：未点击或点击偏差过大。Combo 重置，屏幕触发红色晃动，播放失协音。
- **Combo 进化系统**：
    - **1-10 Combo**：普通蓝色数字。
    - **11-30 Combo**：数字变大并开始发光（外发光效果）。
    - **31+ Combo**：数字变为彩虹色，并伴随持续的粒子拖尾效果。
- **进度指示 (Progress)**：顶部显示进度条，实时反映当前单词组的完成百分比。

### 7.7 音效反馈策略 (Audio Feedback Strategy)

#### A. 基础反馈音效 (General Feedback)
- **正确反馈 (Correct)**：
    - **触发**：完成拼写子环节或最终拼写正确。
    - **音效**：播放 `Harmony` 或 `Win` 音效（明快的 C 大调琶音）。
- **错误反馈 (Incorrect)**：
    - **触发**：选择错误选项或拼写顺序错误。
    - **音效**：播放 `Dissonance` 音效（低频失协音），伴随视觉抖动。

#### B. 节奏挑战音效 (Rhythm Mode Audio)
- **背景音效 (Audio Engine)**：
    - **核心引擎**：使用 Web Audio API 维持稳定的背景节拍，确保在高 BPM 下依然精准。
    - **4/4 拍鼓组**：循环模式为 `Kick (强) - Hi-hat (弱) - Snare (次强) - Hi-hat (弱)`。
    - **动态音色**：
        - **Kick (底鼓)**：低频冲击波 (150Hz -> 0Hz)，提供核心节奏感。
        - **Snare (军鼓)**：中频三角波，增加节奏的清脆感。
        - **Hi-hat (闭嚓)**：高频方波，负责细分节拍。
- **击中反馈 (Hit/Harmony)**：
    - **机制**：点击正确字母时触发。
    - **动态音调**：采用五声音阶 (Pentatonic Scale)，音调随 `Combo` 数增加而阶梯式上升。Combo 越高，音色越清脆（Sine Wave）。
- **阶段成功 (Success/Win)**：
    - **机制**：完成单词或整组挑战时触发。
    - **音色**：明快的 C 大调琶音 (C-E-G-C)，使用三角波营造复古 8-bit 游戏感。

### 7.8 视觉生成策略 (Image Prompt Specification)
- **生成逻辑 (Generation Logic)**：
    - **Step 1 (Prompt Engineering)**：使用 `gemini-3-flash-preview` 根据单词语义生成一段具体的描述性提示词（如：将 "apple" 描述为 "a shiny red apple with a small green leaf"）。
    - **Step 2 (Image Generation)**：将生成的描述性提示词与核心风格关键词结合，调用 `gemini-2.5-flash-image` 生成最终图像。
- **核心风格 (Core Style)**：统一采用“可爱、色彩鲜艳、卡通风格”的矢量插画。
- **提示词结构 (Prompt Structure)**：`[描述性提示词] + [风格关键词] + [构图关键词]`。
- **关键词库 (Keyword Library)**：
    - **风格类**：`cute cartoon style`, `vibrant colors`, `thick lines`, `child-friendly`, `vector illustration`, `soft lighting`.
    - **构图类**：`centered subject`, `simple background`, `white background`, `no text`, `single object`, `high contrast`.
- **内容准则 (Content Guidelines)**：
    - **具象化**：图像必须直观反映单词语义，避免抽象表达。
    - **单一焦点**：优先生成单个物体或清晰的动作，减少背景干扰。
    - **安全性**：严禁出现暴力、恐怖或任何不适合儿童的元素。
    - **无文字**：提示词中必须包含 `no text`，防止 AI 在图片中生成错误的拼写。
- **容错机制**：若图像生成失败或触发配额限制，系统将自动生成基于单词种子的占位 SVG 图标。

## 8. 非功能性需求 (Non-Functional Requirements)

### 8.1 性能要求
- **AI 响应**：文本生成 < 3s，图像生成 < 5s。
- **流畅度**：界面切换动画需保持 60fps，节奏模式延迟 < 20ms。

### 8.2 安全性
- **数据安全**：导出文件需经过 XOR 混淆，防止明文泄露。
- **内容安全**：AI 需过滤敏感、暴力或不适合儿童的词汇。

### 8.3 兼容性
- **设备**：适配主流平板电脑（iPad 等）及桌面浏览器。
- **离线**：已生成的单词数据必须支持 100% 离线练习。

## 9. 风险评估与对策 (Risk Assessment)

| 风险点 | 严重程度 | 对策 |
| :--- | :--- | :--- |
| AI 生成内容不准确 | 中 | 引入用户反馈机制，支持手动修正或重新生成。 |
| 网络连接中断 | 高 | 强化本地缓存机制，支持断点续传。 |
| 浏览器存储空间限制 | 低 | 定期提醒用户清理旧数据或导出备份。 |
| API 额度耗尽 | 中 | 提示用户配置个人 API Key，或提供默认 Key 的降级方案。 |

## 10. 成功衡量指标 (Success Metrics)
- **活跃度**：用户平均每日学习时长 > 15 分钟。
- **留存率**：次日留存率 > 40%。
- **学习效果**：用户在节奏模式下的平均 BPM 提升率。
- **词库规模**：单个用户平均掌握单词量 > 50 个。
